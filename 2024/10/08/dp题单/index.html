动态规划——一生之敌！

动态规划题目已经多次让我在关键比赛/时刻沉默了，真的受不了了，因为 dp 一出不会太难，那么有不会太典，处于一个“差不多”的水平，因此必须得多做！接下来进行整理这类型的题目，并附带标签。

最后想说的一点就是：dp 的算法标签（是什么类型的算法）已经不重要了，重要的是它是 dp，比赛常出，就得会！

前言：

在这里放的题并不是偏难或者说难的那种，而是有好多人都能做出来，自己必须也得做出来的题目。

准备加个 tag：

实现难度：低/中/高

  • 低：即使没写出来，赛后也可以自己很快补出来；
  • 中：赛时没写出来，但思来想去（不看题解代码）又很难实现出来，自己的码力不行；
  • 高：待定（一般可能遇不到高的了，毕竟遇到了也补不了了估计）

题单：

  • 百度之星第一场第二题（过了可能有个铜奖了，实现难度：中）： BD202404110串
  • RAICOM2024 省赛最后一题（实现难度：低）： RC-u5 工作安排

讲解&代码：

BD202404110串

纠结的内心：赛后补题，如果相邻转移的话，上一位和当前位与 110110 没太多关系，牵扯不进来，而如果是用 1111 看看能不能转移到 1010 之类的转移，枚举 33 位数的话又感觉确实有道理，但问题是每 33 位枚举的话，转移的跨越几个位的，那之前位的结果（即 dp 值）怎么求呢？

我趣，说实话，吐个槽，怎么感觉实现起来比 D. Grid Puzzle 还恶心啊。

第一次尝试：写完代码后，发现每次枚举当前位变或不变，有点问题，前面都没有枚举不是很有问题吗？？

第二次尝试前的猜想：其实主要是在想怎么枚举，想到了一个上一位也枚举一下，计算下状态，然后由上一状态转移到当前状态的贡献？虽然感觉比上一次尝试有点道理，但是既然枚举了上一位，咋不继续再枚举一位，即为什么枚举上一位就够了，或者这个对的吗？

下面是真题解：

到了2024.7.31晚上，觉得无聊，想来补题了。先是再看了一遍题解视频，这一次感觉比第一次更有感觉了，于是开始在草稿纸上写了转移方程。当然，我可以解决之前提出的一些奇怪的问题了。

因为我们必须明确改了多少位，因此当前位就是改了还是没改。

没改就是：$f[i][j]=f[i−1][j]$

改了就是：$f[i][j]=f[i−1][j−1]$

而且无论改没改，我们都是已知这一位是什么的，那么讨论第三维度：

$f[i][j][]=f[i−1][j][]$

$f[i][j][]=f[i−1][j−1][]$

其实也就差第三维，但这一位可以强制赋值状态，即最后一维表示的是状态，不是数字0/1，而是状态0/1/2（即最后是0表示状态0，最后是01或1就是状态1，最后是11就是状态2）：

所谓强制赋值状态，就是：如果最后一位是0，显然就是状态0，由上一维哪些状态转移过来呢，就是自动机了；

如果最后一位是1，可能是状态1或2，对每个可能的状态根据自动机对应转移即可。

最后由于卡了空间，我直接转成2维了，习惯这种方法了。

#include<bits/stdc++.h>

using namespace std;

using ll = long long;

constexpr int mod = 998244353;

int main()
{
    cin.tie(nullptr)->sync_with_stdio(false);

    int n, m;
    cin >> n >> m;

    string s;
    cin >> s;
    s = " " + s;

    vector<vector<vector<ll>>> f(2, vector<vector<ll>>(m + 1, vector<ll>(3)));
    f[0 & 1][0][0] = 1;
    for (int i = 1; i <= n; i++)
    {
        for (int j = 0; j <= m; j++)
        {
            // 不改
            int cur = s[i] - '0';
            if (!cur)
            {
                f[i & 1][j][0] = (f[i - 1 & 1][j][0] + f[i - 1 & 1][j][1]) % mod;
            }
            else
            {
                f[i & 1][j][1] = f[i - 1 & 1][j][0];
                f[i & 1][j][2] = (f[i - 1 & 1][j][1] + f[i - 1 & 1][j][2]) % mod;
            }

            // 修改
            if (cur)
            {
                if (j)
                    f[i & 1][j][0] = (f[i - 1 & 1][j - 1][0] + f[i - 1 & 1][j - 1][1]) % mod;
                else
                    f[i & 1][j][0] = 0;
            }
            else
            {
                if (j)  
                {
                    f[i & 1][j][1] = f[i - 1 & 1][j - 1][0];
                    f[i & 1][j][2] = (f[i - 1 & 1][j - 1][1] + f[i - 1 & 1][j - 1][2]) % mod;
                }
                else
                {
                    f[i & 1][j][1] = 0;
                    f[i & 1][j][2] = 0;
                }
            }
        }
    }

    ll res = 0;
    for (int i = 0; i <= m; i++)
        res = (res + f[n & 1][i][0] + f[n & 1][i][1] + f[n & 1][i][2]) % mod;

    cout << res << "\n";
    return 0;
}

蓝桥杯2024年第十五届决赛真题-数位翻转

首先，百分百 dp𝑑𝑝。

可能有点像什么区间dp，或者是搞个前缀和以及翻转的前缀和之类的性质来解决（纯是猜测）。

最后的结果也证实了我的猜测都是错的！

猜测并不是不允许，但是一切要从实际出发，不能以猜测为”根“来思考。

正常思路：

dp 的话一般是区间 [1,n] 或者是前 n 项什么什么的，前者是区间 dp，后者像线性 dp （其实并不重要，因为一切要从实际出发）。如果是区间去思考，其中的一个区间翻转了 x 份，那么拼接其前面的，可能就要讨论是前一个区间的后面几个是和当前区间的前面几个拼接的？那估计复杂度得有 O(N3)了，其次，都讨论其中的一块区间了，如果转换成前 n 项是不是更容易做一些（这一点需要仔细想想，确实如此）？然后就开始思考后者了，仿照答案来定义 dp，那大致就是 $dp[n][m]$：前 n 项，（连续）翻转了 m 次的最大结果。因为不超过 m 次就行，因此最后 O(N) 枚举是不需要什么时间复杂度的。然后就是如何转移了：显然，上一个状态就是 dp[i−1]，就有前 i−1 项是翻转了 m 次还是翻转了 m−1 次（这里已经很接近答案了）。因此转移方程如下：

  • $dp[i][j]=dp[i−1][j−1]+reva[i]$
  • $dp[i][j]=dp[i−1][j]+a[i]$

其实仔细看上面的式子，貌似挺有道理的，但实际上还是有点缺陷：

  • 第一个式子如果最后一项是被翻转的，那么接着翻转的话，貌似并不用这个次数（j−1 次）？如果非要说这个也可以的话毕竟我可以连续的分成多个来翻其实也成立，那最大的不足应该是第二个式子了；
  • 第二个式子前 i−1 项分成 j 份，但是并不影响再和第 i 项凑成连续的，而这个成立的条件是前 i−1 个的最后一个也要被选，如果不被选的话就是原式了。

因此，可以发现，跟最后一个选或不选有一定的关联，因此不妨再加一维表示最后一个数选或不选，那么就很对了，式子略，具体看如下代码：

#include<bits/stdc++.h>

using namespace std;

using ll = long long;

int main()
{
    cin.tie(nullptr)->sync_with_stdio(false);

    int n, m;
    cin >> n >> m;

    vector<ll> a(n + 1), reva(n + 1);
    auto get = [&](int x) -> ll
    {
        ll res = 0;
        while (x)
        {
            res = res * 2 + x % 2;
            x /= 2;
        }
        return res;
    };

    for (int i = 1; i <= n; i++)
    {
        cin >> a[i];
        reva[i] = get(a[i]);
    }
    vector<vector<vector<ll>>> dp(n + 1, vector<vector<ll>>(m + 1, vector<ll>(2)));
    for (int i = 1; i <= n; i++)
    {
        for (int j = 0; j <= min(m, i); j++)
        {
            if (j >= 1)  dp[i][j][1] = max({dp[i - 1][j - 1][0], dp[i - 1][j - 1][1], dp[i - 1][j][1]}) + reva[i];
            dp[i][j][0] = max(dp[i - 1][j][0], dp[i - 1][j][1]) + a[i];
        }
    }

    ll res = 0;
    for (int i = 0; i <= m; i++)    res = max({res, dp[n][i][0], dp[n][i][1]});
    cout << res;
    return 0;
}

D. Grid Puzzle

解题步骤：性质-> dp -> 微调 dp 定义->解决。

赛后补题了16分钟，不知道怎么转移，就转移的方式很难（等会细说我的奇怪做法）。cf现场的时候其实能看出来是dp的，但写不出来就是得多练。

奇怪做法：涂每行的前四个格子有3个操作，转换成二进制就是3个位，但往下面转移的需要知道上面空缺的且后面涂完的方案，这个感觉卡死了，根本求不出来吧，因此，感觉自己的dp定义出错了。。。

大致看了题解了解的思路后，知道怎么定义了，但感觉转移讨论有点复杂，但是不能被这个一直扰乱思绪而不去实现，应该先把转移方程写出来，列出转移式子之后实现一下，再考虑讨论的情况（特殊情况）。

上面可能说了很多废话（可以忽略），下面进行总结：

  • 虽然是中等码力的题目，转移有点混乱又可能需要分类讨论；
  • 但是必须要先明确好 dp 的定义才能继续往后写，这是一切的基础；
  • 定义好 dp 后，接下来是分类讨论，为防止混乱下面是一种思路（因为有特殊情况，但不能跟转移的公式混淆）：
    1. 先根据定义的内容，写出转移方程；
    2. 根据写好的代码去调试样例，找到问题所在；
    3. 如果遇到 dp 定义的可能有些欠缺，可以反过来稍微修改（下面的代码就是经过了修改）。

总结完后，再说些下面具体代码的一个巧妙的修改：

  • $dp[i][1]$：当前行涂一行；
  • $dp[i][2]$：当前行涂1,2列；
  • $dp[i][3]$：当前行涂3,4列。

首先是明确必须得涂，但是会遇到一些问题，这里懂的都懂就不细说了。

而解决这个问题最好的方法，就是修改 $dp[i][1]$ 的定义，因为他们都是明确涂的就是这个东西，不过可以很丝滑的改成”涂完前 i 行且不往下有被涂过的“。这样首先可以保证仍然跟后面 2 个定义是有区分的，且不重不漏；其次可以解决当前如果不需要涂，就可以不涂的情况。因此修改完毕。

#include<bits/stdc++.h>

using namespace std;

void solve()
{
    int n;
    cin >> n;

    vector<int> a(n + 1);
    for (int i = 1; i <= n; i++)    cin >> a[i];

    vector<vector<int>> dp(n + 1, vector<int>(4, n));
    /**
     * 定义3个dp值，彼此不交：
     * 0：涂一行
     * 1：涂1,2
     * 2：涂3,4
     * 依据条件：每行最多只涂2*2,1次
     * 条件是什么？什么叫依据条件？条件凭什么成立？
     */
    /**
     * 更明确的：
     * dp[i][1]:当前行涂完，并且不影响下一行
     * dp[i][2]:当前行涂完，并且下一行的1,2列已经涂完
     * dp[i][3]:当前行涂完，并且下一行的3,4列已经涂完
     */
    dp[0][1] = 0;
    for (int i = 1; i <= n; i++)
    {
        // dp[i][1]
        dp[i][1] = min({dp[i - 1][1] + (a[i] >= 1), dp[i - 1][2] + (a[i] >= 2), dp[i - 1][3] + (a[i] >= 1)});
        if (a[i] <= 2)  dp[i][1] = min(dp[i][1], dp[i - 1][2]); // 有很强的寓意：可以让当前行不填，依托上一行
        // dp[i][2]
        if (a[i] <= 2)  dp[i][2] = min({dp[i][2], dp[i - 1][1] + 1, dp[i - 1][2] + 1});
        if (a[i] <= 4)  dp[i][2] = min({dp[i][2], dp[i - 1][3] + 1});
        // dp[i][3]
        if (a[i] <= 4)  
        {
            dp[i][3] = min(dp[i][3], dp[i - 1][2] + 1);
            if (!a[i])  dp[i][3] = min({dp[i][3], dp[i - 1][1] + 1, dp[i - 1][3] + 1});
        }
        // cout << i << ": " << dp[i][1] << " " << dp[i][2] << " " << dp[i][3] << "\n";
    }
    cout << min({dp[n][1], dp[n][2], dp[n][3]}) << "\n";
}

int main()
{
    cin.tie(nullptr)->sync_with_stdio(false);

    int t;
    cin >> t;

    while (t--)
    {
        solve();
    }
    
    return 0;
}
