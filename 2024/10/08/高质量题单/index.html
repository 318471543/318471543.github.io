精选高质量题目

最近发现了很多好题，属于那种感觉挺典但又不熟悉的领域。也因为上次打 $cf$ 遇到了 $C$ 题觉得，开一个这种类型的题单吧，不写题解，就是写一个这个题目的标签就行了。

感觉下面的题，能放在区域赛（邀请赛）简单题里面或者校赛、蓝桥杯等中等难度偏下的题中，质量值得保证。

如果一个题目下面用白色·标识，表示类似的题型，但并不想把他放在这一类里面，可见非白色·的题质量是多高哇！

思维构造

  • $cf\ div3\ E$（网格构造，码量不大，但思维含量挺高）：E. Cells Arrangement

    • 思维交互题：C. Find a Mine
    • 网格二分+双指针（构造）：F. Custom-Made Clothes

  • 小思维 + 公式推导 = 大思维，然后转换为算法求解：F. Equal XOR Segments

E. Cells Arrangement：

  • 赛时在想，赛后也在想，都想不出来。群友突然发来一个构造方法，看了一下，确实最优。

  • 方法：对角线构造，有一个性质：相邻两个对角线位置，曼哈顿距离为 $2$，每搁一个对角线，曼哈顿距离就会增 $2$，那么就是 $2,4,6\cdots$ 的增长；那奇数怎么办，在一个角落里再搞一个相邻的位置！

  • 反思：像这种构造题，想到了的话发现很简单，想不到的话就不知道从哪找突破口。所以这种题真的很吓人。真没想到就真寄了。

F. Equal XOR Segments：

  • 赛时只能想到小思维，由于抽象的去想怎么求解，忽略了公式的推导，导致根本想不到如何做。实际上必须要将公式列出来才能发现问题的根源。

武汉邀请赛热身赛：

基本想法：单点的修改可以 O(1)得到贡献，一个巧妙的思考方式是按时间顺序处理答案，但继续思考下去的话很难深入，但必须要深入。

深入：一点的影响（即第 i 秒第 j 列），会导致从 i−j+1 开始的所有照片都受到影响（差分区间修改单点加）。然后抽象出每个摄影都是从第 x 秒开始的，差分数组做一遍前缀和之后，得到的就是每一秒开始的照片有多少被修改的点。

继续深入：前缀和之后就是答案，为什么是这样定义的？因为我们最初想的就是将某一区间全部加 1 或全部减 1，这就是我们前缀和得到的东西。

B. Countless Me：

赛后补题：觉得二进制位从高到低枚举是有道理的，但是又要分成 n 份，难道是要换成 n 进制思想吗？又没有道理，实际上是：这 n 个数，拆分成二进制的形式，每一位都纵向考虑，即考虑这一位时是同时考虑 n 个的，只要后面的能表示 sum，这一位的所有 n 个数就都不用为 1。

细节： 这一位如果要填 1，需要计算填多少个 1，即 sum>>i，但是又不能超过 n！会有超过 n 个的情况，因为后面的位数都是用乘法（即 n 个数）来考虑的，因此还需要与 n 取最小值。

反思总结：补完题后，其实核心代码就只有 4 行，赛时却花了 2 小时。

K. Party Games：

赛后总结：赛后很多高手都一眼看出了跟 mod 4 有关，因此前 i 项连续异或和应该还是得记一下。

下面给出 dp 和 SG 的两种打表的做法：

dp：

#include<bits/stdc++.h>

using namespace std;

using ll = long long;

int main()
{
	cin.tie(nullptr)->sync_with_stdio(false);

	int n = 100;
	vector<int> pre(101);
	for (int i = 1; i <= n; i++)	pre[i] = pre[i - 1] ^ i;

	int dp[102][102] {};
	for (int  l = n; l >= 1; l--)
	{
		for (int r = l;r <= n; r++)
		{
			if (pre[l - 1] == pre[r])	dp[l][r] = 0;
			else	dp[l][r] = !dp[l + 1][r] || !dp[l][r - 1];
		}
	}
	for (int i = 1; i <= n; i++)	
		cout << i << " " << dp[1][i] << "\n";
	return 0;
}

SG 打表：

#include<bits/stdc++.h>

using namespace std;

using ll = long long;

int main()
{
	cin.tie(nullptr)->sync_with_stdio(false);

	int pre[101] = {};
	for (int i = 1; i <= 100; i++)	pre[i] = pre[i - 1] ^ i;
	function<int(int, int)> SG = [&](int l, int r) -> int
	{
		if ((pre[r] ^ pre[l - 1]) == 0)	return 0;
		else if (l == r)	return 1;
		
		set<int> S;
		if (l + 1 <= r)	S.insert(SG(l + 1, r));
		if (l <= r - 1)	S.insert(SG(l, r - 1));

		for (int i = 0; i < 100; i++)
			if (!S.count(i))
				return i;
		return -1;
	};

	for (int i = 1; i <= 20; i++)
		cout << i << " " << SG(1, i) << "\n";
	return 0;
}
